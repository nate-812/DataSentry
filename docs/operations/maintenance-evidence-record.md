# 维护窗口证据记录模板

本文档用于 M9 暴露面收口、账号治理、systemd 调整和后续维护窗口的人工证据记录。它只记录状态、ID、脱敏摘要和失败项，不替代现场只读查询。

## 使用边界

- 不记录真实 secret、Cookie、私钥、完整连接串或业务敏感样本值。
- 不把历史 smoke 包装为当前事实；每次维护窗口都必须记录当前命令结果和时间。
- 不执行未确认的生产写操作；systemd、账号、云安全组、数据库权限和业务配置变更都需要用户再次确认。
- 不自动修改云安全组，不开放生产写 Runbook，不引入自动重启、自动补数、自动改配置或自动 Savepoint 恢复。
- 不开云端实例时只预填计划信息，不填写历史结果为通过。

## 空白记录

```text
维护窗口：
操作者：
云端实例：
Git commit：

变更项：
变更前状态：
用户确认：
执行动作：
变更后验证：
失败项：
回滚动作：
Incident id：
Inspection id：
未验证项：
secret 处理确认：未打印、未落盘到仓库、未写入日志或 RCA
```

## 填写规则

- `维护窗口` 记录日期、开始时间、结束时间和时区。
- `Git commit` 记录云端实际运行的 DataSentry commit，不记录本地未部署提交。
- `变更前状态` 只记录只读确认结果、命令摘要和脱敏错误；不要提前填“正常”。
- `用户确认` 记录确认时间、确认范围和允许执行的具体动作。
- `执行动作` 只记录已确认动作；跳过项写明“未执行”和原因。
- `变更后验证` 记录 health、smoke、固定只读巡检和组件级回归结果。
- `失败项` 写清失败命令、失败状态和下一步，不用历史成功结果覆盖当前失败。
- `回滚动作` 只记录本轮实际回滚；没有回滚时写“未执行回滚”。
- `Incident id` 和 `Inspection id` 只记录 ID，不粘贴 RCA 全文、业务样本或 secret。
- `未验证项` 必须显式写出原因，例如“云端实例未打开”或“用户暂缓该组件”。

## 复核清单

- [ ] 已确认本轮是否打开 `data1`。
- [ ] 已确认 DataSentry API 和 Alertmanager proxy 的 `systemctl is-enabled` 与 `systemctl is-active`。
- [ ] 已记录监听地址、云安全组事实和组件 health。
- [ ] 已记录 `datasentry ops preflight`、`deployment-check`、`alert-smoke` 或暂缓原因。
- [ ] 已记录真实 K 线只读巡检 Inspection id 或暂缓原因。
- [ ] 已确认日志、通知、RCA、状态文档和 Git diff 不包含真实 secret。
- [ ] 已确认未开放生产写 Runbook、自动重启、自动补数、自动改配置或自动 Savepoint 恢复。
