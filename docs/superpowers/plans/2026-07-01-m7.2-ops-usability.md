# M7.2 Ops Usability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make local live-smoke operations safer and easier by adding preflight secret checks, clearer missing-secret output, and a no-secret operations guide.

**Architecture:** Add a small `datasentry.ops` module that inspects `TargetCatalog` metadata and environment presence without reading secret values. Wire it into the Typer CLI as `datasentry ops preflight`, and keep `inspection run` behavior unchanged except for safer causal error details already exposed by domain errors.

**Tech Stack:** Python 3.12, Typer, Pydantic v2, pytest, Ruff, mypy, Markdown docs.
